# 提案 Intent — Speed-ordered turns

## 這是什麼

把戰鬥從「玩家永遠先攻」改為「根據 SPD 屬性排序行動順序」。
高 SPD 者先動，相同 SPD 時玩家優先（避免速度同步時的負面體驗）。

## 為誰而做

- **現有玩家**：得到符合 RPG 經典直覺的戰鬥節奏
- **未來內容**：v0.2+ 會加入更多怪、含「快速但脆弱」的敵人類型，
  例如蜘蛛 (SPD 9) 應該比玩家 (SPD 5) 先攻；目前在 MVP 卻被反邏輯地後攻
- **作者**：練習在 Zustand store 上做戰鬥狀態機重構

## 解決什麼問題

`docs/design.md` §4.2 寫的 pseudocode 明確指出「根據速度排序角色 → 行動順序」，
但 MVP 為了簡化先做「玩家固定先攻」。這個 cycle 補上 design 承諾，
讓速度屬性 (SPD) 真正影響戰鬥節奏。

## 範圈

- ✅ 1v1 戰鬥的順序排序（player vs single enemy）
- ❌ 不處理「同一回合內中途暈眩/減速」（buff/debuff 系統屬未來）
- ❌ 不處理多人隊伍（屬 v0.3 #9）
- ❌ 不改傷害公式
- ❌ 不改 AI（issue #5 範圍）
