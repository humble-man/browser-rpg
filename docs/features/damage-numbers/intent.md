# 提案 Intent — Floating damage numbers

## 這是什麼

戰鬥中當任一方受到傷害時，在被攻擊者頭上浮動顯示傷害數字 (~1 秒)：
- 普通傷害：白色數字飛出 → 上升 → 淡出
- 爆擊：黃色 + 較大 + 「!」後綴
- 治癒：綠色 + 「+」前綴

```
       💥 24
        ↑
    [🐉 Dragon]
       HP: 130 / 180
```

## 為誰而做

- **戰鬥中的玩家**：直觀看到「我這發打了 N 點」、不必盯 log
- **爆擊偵測**：用顏色立刻分辨爆擊
- **配合 cycle 14 SE**：視覺 + 聽覺雙重 hit feedback
- **作者**：練習 React 動畫 lifecycle + key-driven re-mount

## 解決什麼問題

戰鬥目前只用「shake animation + log line」呈現傷害。
玩家連點攻擊時 log 訊息會被前面動畫蓋掉，看不清具體傷害值。
本 cycle 加浮動數字 = 即時、明確、可分辨爆擊/治癒的視覺回饋。

## 範圈

- ✅ 玩家攻擊命中敵人 → 敵人頭上浮數字
- ✅ 敵人攻擊命中玩家 → 玩家頭上浮數字
- ✅ 爆擊用不同顏色 + 大小 + 「!」
- ✅ 治癒 (玩家自己用 heal skill / item) → 玩家頭上綠色 「+N」
- ✅ 動畫 ~1 秒、上升 + 淡出
- ❌ 不加 miss / dodge 訊息（沒這機制）
- ❌ 不加 damage type color（火/冰/物理等，沒這機制）
- ❌ 不加粒子特效
- ❌ 不改 lastDamage 資料結構（複用既有 state）
